
# Java Concurrency Programming

## 1. Principle

volatile实现原则：

	1. Lock前缀指令会将处理器缓存回写到内存。
	1. 一个处理器的缓存回写到内存，会使得其他处理器的缓存无效。

synchronized同步有三种形式

1.   普通同步方法，锁当前实例对象。
2.   静态同步方法，锁当前Class对象。
3.   同步方法块，锁括号里的配置对象。

为了减少锁的获得与释放带来的性能损耗，引入了“偏向锁”和“轻量级锁“，从高到低有4种状态，分别是：无锁状态、偏向锁状态、轻量级锁状态、重量级锁状态。其会随着竞争情况逐渐升级。

偏向锁（Biased Lock）：查看对象头是否有偏向锁，如果没有，把Mark Word设置为1，如果有，则指向当前线程。
